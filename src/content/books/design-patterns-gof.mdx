## Por que ler

E o catalogo original de 23 design patterns que todo livro e framework referencia. A linguagem usada e C++/Smalltalk, mas os CONCEITOS sao universais.

## Os patterns mais relevantes para web developers

**Creational:**
- **Factory Method:** Cria objetos sem especificar a classe exata. Usado em ORMs (model.create()), frameworks de teste (createMock()).
- **Singleton:** Instancia unica global. Usado em conexoes de banco, loggers, stores.
- **Builder:** Constroi objetos complexos passo a passo. Usado em query builders (Knex), request builders.

**Structural:**
- **Adapter:** Converte interface incompativel. Usado quando voce wrapa uma API legada em uma interface moderna.
- **Decorator:** Adiciona responsabilidades sem heranca. TypeScript decorators (`@Injectable()`, `@Controller()`).
- **Proxy:** Controla acesso a um objeto. Usado em JS Proxies, lazy loading, caching.

**Behavioral:**
- **Observer:** Notifica mudancas. Event Emitters, React state, PubSub.
- **Strategy:** Troca de algoritmo em runtime. Formas de pagamento, validacoes.
- **Chain of Responsibility:** Passa request por uma cadeia. Express middleware!
- **Iterator:** Percorre uma colecao sem expor estrutura. `for...of` em JS.
